# 食养蒸馏方法论 v3.0 —— 融合版

> 将国家卫生健康委发布的官方食养指南与专业书籍完整蒸馏为AI科普对话Skill

创建者：王润圆（中国注册营养师，昆明医科大学营养与食品卫生学硕士）
工具：WorkBuddy
版本：v3.0（融合 book-to-skill 框架）

---

## 一、方法论定位与核心理念

### 1.1 双重定位

本方法论融合两条蒸馏路线：

| 路线 | 来源 | 适用场景 |
|------|------|---------|
| **食养指南蒸馏** | 国家卫健委官方指南 | 慢性病食养科普Skill |
| **书籍蒸馏** | book-to-skill 框架 | 专业书籍→结构化Skill |

### 1.2 核心理念

**科普性质，非教材性质** —— 自然贴地气，同时保持精准。

- **完整蒸馏**：数据必须忠实于原文
- **贴地气风格**：像社区营养师在和你聊天
- **出处可溯**：每个数据点都能回溯到原文
- **提取结构，而不是总结**：捕捉命名框架、精准表述、反模式，而不是章节复述
- **密度优先**：1000 token 的精炼胜过 10000 token 的摘录
- **实践者语气**：写 "用 X 做 Y"，而不是 "书中提到 X"

### 1.3 适用范围

- 国家卫生健康委发布的《成人XXX食养指南》系列
- 营养学/医学/健康类专业书籍
- 任何需要结构化知识蒸馏为AI Skill的文档

## 二、融合知识架构

### 2.1 双层知识单元

| 层级 | 名称 | 来源 | 用途 |
|------|------|------|------|
| KPK | 科普知识点 | 食养蒸馏 | 指南数据蒸馏的最小完整单元 |
| Framework | 框架模型 | book-to-skill | 书籍核心方法论提炼 |

### 2.2 KPK 定义（食养指南场景）

KPK = **科普知识点**，是从官方指南中蒸馏出的最小完整知识单元。

每个 KPK 包含以下7个字段：

| 字段 | 说明 | 示例 |
|------|------|------|
| **KPK ID** | 编号，格式 KPK-XX | KPK-01 |
| **核心知识** | 一句话概括本知识点 | 能量适宜蛋白优质，调控脂肪和碳水 |
| **适用人群** | 本知识点针对的目标人群 | 脑卒中恢复期及后遗症期患者 |
| **食养建议** | 具体可操作的建议 | 蛋白质≥1.0g/kg/d，感染/压疮者1.2-1.5g/kg/d |
| **食材/方剂** | 推荐的具体食物或中医方剂 | 鱼类300-500g/周，瘦肉约50g/d |
| **注意事项** | 使用限制和禁忌 | 肾功能不全者蛋白质需调整 |
| **关联KPK** | 与其他知识点的关联 | KPK-09（食物选择），KPK-10（交换表） |

### 2.3 Framework 定义（书籍场景）

Framework = **可复用的框架/模型**，是从专业书籍中蒸馏的核心方法论单元。

| 字段 | 说明 |
|------|------|
| **名称** | 精确保留作者命名（如"控糖顺序法"，而非"先吃菜再吃饭"） |
| **适用场景** | 什么情况下使用 |
| **操作方法** | 步骤化说明 |
| **原理** | 为什么有效 |
| **反模式** | 常见错误用法 |
| **关联章节** | 出自哪一章 |

## 三、四模式工作流

根据任务类型选择模式：

| 模式 | 触发条件 | 工作流 |
|------|---------|--------|
| **完整蒸馏**（默认） | 提供完整源文件 | 全部步骤 |
| **快速分析** | 用户说"先分析"、"just extract" | 步骤 0→1→1.5→2→3，输出分析报告后停止 |
| **从已有分析生成** | 有现成的分析笔记 | 跳过步骤 0-3，从步骤 4 开始 |
| **增量更新** | 在已有 Skill 上追加新内容 | 执行更新/合并工作流 |

### Step 0：范围检查

检查输入参数：
- 如果无参数 → 停止，提示需要文档路径
- 如果指向已有Skill目录 → 自动切换到 **增量更新模式**

### Step 1：输入验证

确认至少一个支持的文档存在：
- 指南场景：`.pdf`，支持 `pdftotext -layout` 提取
- 书籍场景：`.pdf`, `.epub`, `.docx`, `.txt`, `.md`, `.html`

### Step 1.5：内容类型识别

询问用户文档类型（影响提取方法选择）：
1. **技术型** — 含表格、代码、公式（如学术论文、数据密集型书籍）
2. **文字型** — 以文字为主（如管理类、叙事非虚构）
3. **不确定** — 使用快速方法，质量受限时发出警告

### Step 2：文本提取

**指南PDF提取**：
```bash
pdftotext -layout guide.pdf main_guide.txt
pdftotext -layout guide_qa.pdf qa_guide.txt
```

**书籍文档提取**：
- Markdown/HTML → 直接读取
- PDF → pdftotext（文字型）或 Docling（技术型）
- EPUB/DOCX → 使用适当转换工具

**验证**：检查字符数，抽查关键数据

### Step 2.5：成本估算（大文档时）

展示预计 token 消耗，征求用户同意后再继续：

```
📖 源文件: <文件名>
📄 页数: ~N | 字数: ~N | 估算token: ~NK

💰 预估成本:
   输入（读取+提示）: ~NK tokens
   输出（Skill文件生成）: ~NK tokens
   合计: ~NK tokens

📁 待生成文件:
   SKILL.md + KPK/Framework + 食谱/章节 + 术语表 + 速查表

➡ 继续？（或输入"只分析"先预览）
```

### Step 2.6：REPL式访问（>50k tokens 时）

对于超过 50k tokens 的大文档，使用 grep/sed 精确定位章节，而非一次性加载全文：

```bash
# 查找章节偏移
grep -n -E "^(第?[一二三四五六七八九十]+章|Chapter|附[录件])" full_text.txt

# 仅提取需要的章节
sed -n '<start>,<end>p' full_text.txt

# 验证关键概念是否存在
grep -c -i "中医\|辨证\|交换表" full_text.txt
```

### Step 3：结构分析

**指南场景**：识别食养原则条数、附录数量、中医证型

**书籍场景**：识别标题、作者、章节结构、核心主题

### Step 4：确认目标

询问用户：这个 Skill 应该帮助做什么？
1. **科普对话** — 面向大众的问答式食养助手
2. **框架参考** — 供专业人士查阅的方法论工具
3. **两者都要** — 默认选项

## 【指南蒸馏分支】

### 指南 Step 2：分批蒸馏KPK

**原则**：分批蒸馏，逐步验证，最后合并

| 批次 | 文件 | 内容 | 蒸馏要点 |
|------|------|------|---------|
| 第1批 | kpk_principles.md | 食养原则章节 | 每条原则→1个KPK，含完整数据点 |
| 第2批 | kpk_appendix_1.md | 附录前半部分 | 食物选择表、交换表等大表格必须完整 |
| 第3批 | kpk_appendix_2.md | 附录后半部分 | 中医食养方、筛查工具、术语表等 |
| 第4批 | kpk_disease_qa.md | 疾病定义+问答版 | 流行病学数据、中医认识、参考文献 |

**蒸馏提示词模板**：
```
你是一个营养学知识蒸馏专家。请从以下食养指南文本中，按KPK框架蒸馏知识点。

源文件：[文件路径]
行范围：[起始行-结束行]

要求：
1. 每个KPK包含6个字段：KPK ID、核心知识、适用人群、食养建议、食材/方剂、注意事项、关联KPK
2. 所有数值必须与原文完全一致
3. 表格数据必须完整蒸馏
4. *标记（食药物质）和#标记（新食品原料）必须保留
5. "注"部分不可省略

输出：写入 [目标文件路径]
```

### 指南 Step 3：食谱数据提取

**规模**：7地区 × 4季节 × 3能量级别 = 84套全天膳食食谱

**并行策略**：按地区分组，多代理并行提取

**食谱提取提示词模板**：
```
你是一个数据提取专家。请从食养指南文本中提取[地区]的所有四季食谱数据。

每个地区有4季×3个能量级别 = 12套食谱

输出格式（每套食谱）：
### [地区] [季节]食谱[N]（总能量约[X]kcal）

| 餐次 | 食谱 |
|------|------|
| 早餐 | 具体内容（含食材克数） |
| 午餐 | 具体内容 |
| 晚餐 | 具体内容 |
| 油、盐 | 全天总用量：植物油Xg，盐＜Xg |

> 注：1.蛋白质/碳水/脂肪克数及占比 2.*为食药物质
```

### 指南 Step 4：合并为知识库

```bash
cat kpk_*.md recipes_*.md > knowledge_base.md
```

检查 KPK 编号连续性，无遗漏无重复。

### 指南 Step 5：编写系统提示词（system_prompt.md）

这是最核心的文件，包含以下7个模块：

#### 模块1：四维配置

```yaml
知识维度:
  知识库: knowledge_base.md
  食谱库: recipes_data.md + recipes_overview.md

风格维度:
  基调: 专业但平易近人的社区营养师
  语言: 说人话 + 给具象量 + 指误区
  翻译规则: [见通俗翻译规则]

应答维度:
  模式: 通俗版VA（6层结构）
  场景: [见场景化应答]

人格维度:
  优先级: [如脑卒中的吞咽安全永远第一]
  反模式: [见反模式列表]
```

#### 模块2：通俗翻译规则

| 学术术语 | 通俗翻译 |
|---------|---------|
| 脑卒中 | 中风 |
| 缺血性卒中 | 脑血管堵了 |
| 吞咽障碍 | 咽东西费劲/喝水呛 |
| 误吸 | 吃喝的东西进了气管很危险 |
| 肌少症 | 肌肉"缩水" |
| 优质蛋白 | "好蛋白" |

#### 模块3：通俗版VA应答模式

```
通俗版VA六层结构：
1. 一句话结论     → 直接回答问题
2. 生活化比喻/解释 → 用日常语言解释原理
3. 精准数据       → 给出指南原文数据
4. 实操方案       → 具体怎么做
5. 常见误区       → 指出大众易犯的错误
6. 来源           → 引用指南出处
```

#### 模块4：场景化应答

根据指南特点设计5-7个高频咨询场景，每个场景包含：
- 触发条件
- 应答策略
- 安全提醒
- 优先级排序

#### 模块5：关键数据速查表

| 项目 | 推荐量 | 通俗理解 |
|------|--------|---------|
| 食盐 | <5g/天 | 一个啤酒瓶盖 |
| 烹调油 | ≤25g/天 | 两汤匙封顶 |
| 小腿围 | <34cm | 比矿泉水瓶粗不了多少就得当心 |

#### 模块6：用户画像检测

| 画像 | 特征 | 应答策略 |
|------|------|---------|
| 患者本人 | 直接问自己情况 | 先评估→再建议→给方案 |
| 家属照护者 | 问家人怎么吃 | 强调实操，给分步骤指导 |
| 高危人群 | 预防为主 | 生活方式指导+定期筛查 |
| 医护人员 | 问专业细节 | 提供精确数据+文献引用 |

#### 模块7：反模式列表

| # | 反模式 | 说明 |
|---|--------|------|
| 1 | 不替代医生诊断 | 明确说"建议就医" |
| 2 | 不编造数据 | 只用指南原文数据 |
| 3 | 不忽视吞咽安全 | 脑卒中特有，吞咽安全永远第一 |
| 4 | 不推荐单一食疗方 | 强调辨证选用 |
| 5 | 不忽视急性期 | 明确本指南适用于恢复期 |
| 6 | 不复制原文 | 总是综合、提炼、提取信号（来自book-to-skill） |

## 【书籍蒸馏分支】

### 书籍 Step 7：生成章节摘要

每个章节摘要文件 **800-1,200 tokens**，按以下结构：

```markdown
# 第N章：<完整标题>

## 核心观点
<1-2句话：本章最重要的一个知识点>

## 框架/模型引入
- **<框架名称>**：<精确表述 — 保留作者的命名>
  - 何时使用：<具体场景>
  - 如何操作：<步骤或标准>

## 关键概念
- **<术语>**：<一句话精确定义>
(5-10个本章最重要的术语)

## 反模式
- **<避免什么>**：<为什么失败>

## 核心要点
1. <可操作的洞见>
2. <可操作的洞见>
3. <可操作的洞见>
(3-7条)

## 关联章节
- **第N章**：<为什么关联>
```

### 书籍 Step 8：生成支撑文件

#### glossary.md（术语表）
- 按字母/拼音排序
- 格式：`**术语** — 定义（第N章）`
- 最多 1,500 tokens

#### patterns.md（模式库）
- 所有具体技术、设计模式、算法
- 格式：`## 模式名称\n**何时使用**: ...\n**如何操作**: ...\n**权衡**: ...`
- 最多 2,000 tokens

#### cheatsheet.md（速查表）
- 决策表、对比矩阵、快速参考规则
- 内容如一张打印页

### 书籍 Step 9：生成主 SKILL.md

**关键预算**：SKILL.md 正文控制在 **4,000 tokens 以内**

结构：
```markdown
---
name: <skill_name>
description: "基于"<完整标题>"的知识库。当需要运用<作者>的<核心主题>框架、研究该书或引用其概念时使用。"
---

# <完整标题>
**作者**: <作者> | **章节**: N章 | **生成日期**: YYYY-MM-DD

## 如何使用

- **无参数** — 加载核心框架
- **指定主题** — 搜索并读取相关章节
- **指定章节** — 加载特定章节
- **浏览** — 问"有哪些章节？"

---

## 核心框架
<!-- 最重要内容放前面，约2,000 tokens -->

---

## 章节索引
| # | 标题 | 核心框架 |
|---|------|---------|
| [ch01](chapters/ch01-xxx.md) | 标题 | 框架1, 框架2 |

## 主题索引
<!-- 按拼音排序 -->
- **术语** → chN

## 支撑文件
- [术语表](glossary.md)
- [模式库](patterns.md)
- [速查表](cheatsheet.md)
```

## 五、增量更新工作流

当在已有 Skill 上追加新内容时：

### 5.1 读取现有结构
- 读取现有 SKILL.md 的章节索引、主题索引
- 找到最大章节编号

### 5.2 匹配内容
- 已有章节的新增内容 → 合并更新
- 全新章节 → 创建新文件，编号接续

### 5.3 合并支撑文件
- 术语表合并并重新按拼音排序
- 模式库追加新内容
- 速查表集成新增内容

### 5.4 重生成主 SKILL.md
- 更新章节计数
- 核心框架中融入新增内容（保持 <4,000 tokens）
- 章节索引和主题索引追加新条目

## 六、质量规则
